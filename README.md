This is a project for a dual credit engeering class. The attached CSV file is the training data I gathered for the password algorithm. That alogrithm will analize all of the inputed data and create a four digit code based on the desired weight percentile. The two batch spripts are used to get around the requirment of simulink to run multiplul processes of matlab at the same time, with the down side that a dedixated arduino is needed for this work around. The bat effectivly just creates a spoofed second iteration of matlab that will not actually fully load so the workspaces don't skink and mess something up. The unit ID of the spoofed matlab instance is saved so the second batch file will only close the spoofed instance and not the primary script. The Keypad Script was used as the data collection method for the csv file. Displaytest is the main script and the only one that needs to be executed once everything is wired up.
